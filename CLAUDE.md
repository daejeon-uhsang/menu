# 대전어상 메뉴 프로젝트

## 프로젝트 개요
대전어상(大廃漁床) 전통주 전문점의 메뉴판 관리 시스템.
GitHub에 푸시하면 GitHub Pages를 통해 손님용 메뉴 웹페이지와 인쇄용 A4 페이지가 자동 반영됨.

- GitHub 레포: `daejeon-uhsang/menu`
- 인스타그램: `@daejeon_uhsang`

---

## 파일 구조

| 파일 | 역할 |
|---|---|
| `menu.json` | **메뉴 데이터 원본 (유일한 수정 대상)** |
| `index.html` | 손님용 웹 메뉴 + 인쇄용 12페이지 레이아웃 (menu.json을 fetch로 로드) |
| `대전어상_메뉴관리.html` | 관리자용 GUI 편집 툴 |
| `대전어상_다운로드.html` | 정적 다운로드 파일 — menu.json과 연동 안 됨, 별도 관리 |
| `대전어상_메뉴관리_백업_20260411.html` | 백업 파일, 수정 불필요 |

**메뉴 추가·수정은 항상 `menu.json`만 편집한다.**

## 메뉴 항목(item) 필드

```json
{
  "id": "고유ID (영문+숫자, 중복 절대 불가)",
  "name": "메뉴명",
  "icon": "이모지 (없으면 빈 문자열)",
  "spec": "도수·용량·특이사항 (예: 13% · 500ml)",
  "badge": "new | sig | award | limited | hot | (없으면 빈 문자열)",
  "badgeText": "뱃지에 표시할 텍스트 (예: 신메뉴, 시그니처)",
  "desc": "설명 문구",
  "priceBottle": "병 가격 (예: 39,000) — 잔술 없으면 여기에만",
  "priceGlass": "잔술 가격 (예: 9,000) — 병 판매 없으면 여기에만"
}
```

**badge 종류:**
- `new` → 신메뉴 (빨간계열)
- `sig` → 시그니처
- `award` → 수상·대표
- `limited` → 한정판매
- `hot` → 화제·이슈

## 인쇄 레이아웃 (12페이지 A4)

`index.html`의 `buildPrintPages()` 함수가 menu.json에서 동적으로 생성.
CSS 클래스 `pg2`~`pg12`로 각 페이지 폰트·여백 조정.

| 페이지 | 내용 | CSS 클래스 | 비고 |
|---|---|---|---|
| 1 | 소개 | pi-page | |
| 2 | 주안상 (f1) | pg2 | |
| 3 | 소안상 + 궁중 디저트 (f2+f3) | pg3 | |
| 4 | 샘플러 + 칵테일 (f4+s3) | pg4 | |
| 5 | 조선 하이볼 (s1) | pg5 | 2단 |
| 6 | 마스브루어리·탁주 (s4) | pg6 | |
| 7 | 맑은 술·약주·청주 (s6) | pg7 | |
| 8 | 증류식 소주 1~8번 (s7) | pg8 | 2단, sliceEnd:8 |
| 9 | 증류식 소주 9~15번 (s7) | pg9 | 2단, sliceStart:8 |
| 10 | 프리미엄 잔술 (s8) | pg10 | |
| 11 | 브랜디 + 과실주 (s9+s11) | pg11 | |
| 12 | 조선의 위스키 (s10) | pg12 | 2단 |

**인쇄 페이지 폰트 크기 조정 원칙:**
- 항목이 늘어나면 `pg[N] .piname`, `.pispec`, `.pidesc`, `.pb` 폰트 줄이기
- 항목이 줄어들면 반대로 키우기
- `@media print { }` 블록 안에 위치

## 작업 규칙

### 메뉴 추가 시
1. `menu.json`에서 해당 섹션 `items` 배열에 항목 추가
2. `id`는 영문+숫자 조합, 기존 ID와 중복 불가
3. 커밋 메시지: `[섹션명]: [메뉴명] 신메뉴 추가 (가격)`
4. `git push origin main`

### 메뉴 수정 시
- 가격, 설명, 뱃지 변경은 menu.json 해당 item 수정 후 커밋·푸시

### 인쇄 레이아웃 변경 시
- `index.html`의 `PAGE_GROUPS` 배열과 CSS `pg[N]` 클래스 동시 수정
- 페이지 수 바뀌면 `addPageShell`의 `/ 12` 숫자도 변경

### 절대 하지 말 것
- `대전어상_다운로드.html` 을 인쇄 용도로 쓰지 말 것 (menu.json 미연동)
- `id` 중복 생성 금지
- `menu.json` 외 파일을 메뉴 수정 경로로 쓰지 말 것

## 배포 방식
- `main` 브랜치에 push → GitHub Pages 자동 배포 (수분 내 반영)
- 브라우저 캐시로 즉시 안 보일 수 있음 → Cmd+Shift+R (강력 새로고침)
- `index.html`은 먼저 `./menu.json`을 fetch, 실패 시 GitHub raw URL로 폴백
